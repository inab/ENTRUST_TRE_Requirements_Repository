## Versioning Policy

This repository follows a milestone-anchored semantic versioning scheme:

`<project-milestone>.<minor>.<patch>`

- **Project milestone** versions represent governance-approved baselines
- **Minor** versions capture approved clarifications and corrections
- **Patch** versions capture non-substantive fixes

Historical versions SHALL NOT be modified once released.
